# Study: Hidden Triggers on the Road — Backdoor Attacks in Traffic Sign Recognition
2026-04-13

---

## 1. Overview

This study documents the techniques, design decisions, and implementation process behind the
HTOTRBAITSR research project. The project investigates how backdoor attacks can compromise a
deep learning based traffic sign recognition system and evaluates the effectiveness of lightweight
defence strategies.

The work is grounded in the publicly available German Traffic Sign Recognition Benchmark (GTSRB)
dataset and uses a LeNet convolutional neural network as the base model. Three attack strategies
and four defence mechanisms are implemented and evaluated against each other.

---

## 2. Background and Motivation

Traffic sign recognition (TSR) is a safety-critical component of autonomous driving and advanced
driver assistance systems (ADAS). Models trained on image datasets must operate reliably in
unpredictable real-world conditions. Research has demonstrated that neural networks are vulnerable
to backdoor attacks — a class of adversarial manipulation where an attacker embeds a hidden
trigger pattern into the training data. The model then behaves normally on clean inputs but
produces attacker-specified outputs whenever the trigger appears.

In the context of TSR, a backdoor attack could cause a model to misclassify a Stop sign as a
Speed Limit sign when a small sticker (the trigger) is applied to it. This has clear implications
for vehicle safety.

The core research question driving this project is:

> How vulnerable are traffic sign recognition models to backdoor attacks, and how effective are
> lightweight defence strategies at mitigating them without significantly reducing classification
> accuracy?

---

## 3. Dataset

**German Traffic Sign Recognition Benchmark (GTSRB)**

- Over 50,000 traffic sign images across 43 classes
- Pre-processed to 32x32 pixel resolution and stored as pickled numpy arrays (train.p, test.p)
- Significant class imbalance: the most frequent class has over 2,000 samples while the least
  frequent has under 200
- Split used: 80% of train.p for training, 20% held out for validation, test.p used for final
  evaluation

The pickled format was adopted from the reference implementation at
github.com/hparik11/German-Traffic-Sign-Recognition, which provides pre-resized images compatible
with LeNet input.

---

## 4. Model Architecture: LeNet

The LeNet convolutional neural network was chosen as the base model. It was originally designed
for handwritten digit recognition but scales well to small image classification tasks like GTSRB.

Architecture (ported to PyTorch from the TensorFlow reference):

```
Input (3 x 32 x 32)
  -> Conv2d(3, 32, 5, padding=2) -> ReLU -> MaxPool2d(2)
  -> Conv2d(32, 64, 5, padding=2) -> ReLU -> MaxPool2d(2)
  -> Flatten -> FC(64*8*8, 512) -> ReLU -> Dropout(0.5)
  -> FC(512, 256) -> ReLU -> Dropout(0.5)
  -> FC(256, 43) -> Softmax
```

**Why LeNet:**
- Lightweight enough to run without specialised hardware
- Simple enough to analyse for backdoor research (the penultimate layer is well studied)
- Matches the reference implementation, enabling direct comparison
- The 256-dimensional penultimate layer provides a useful feature space for defence analysis

**Training configuration:**
- Optimiser: Adam (lr=0.001, weight decay=1e-4)
- Scheduler: CosineAnnealingLR over 30 epochs
- Loss: CrossEntropyLoss
- Preprocessing: per-channel histogram equalisation, normalisation to [-1, 1], random rotation
  and affine augmentation during training

**Expected clean accuracy:** >95% on the GTSRB test set (reference repo achieves ~97%)

## 5. Attack Techniques

### 5.1 BadNets (Gu et al., 2019)

BadNets is the foundational backdoor attack. A small pixel patch (e.g. 3x3 white square) is
stamped at a fixed corner of a selected subset of training images. Those images are then relabelled
to the attacker's target class. The model learns to associate the patch with the target class.

**Implementation:**
- Trigger: fixed colour pixel patch at a configurable corner (default: bottom-right)
- Poison rate: fraction of training samples replaced with triggered, relabelled versions
- Key hyperparameters: trigger_size, trigger_position, trigger_color, poison_rate, target_class
- Primary ablation variable: trigger_size (2, 3, 5, 7 pixels)

**Why it works:** The trigger is a strong, unambiguous visual feature. The model has no incentive
to ignore it during training because it consistently co-occurs with the target label.

**Expected results:** >80% attack success rate (ASR) at 10% poison rate, consistent with Gu et al.

---

### 5.2 Blended Injection (Chen et al., 2017)

Blended injection uses a semi-transparent reference pattern (e.g. random noise or a fixed image)
alpha-blended over each poisoned sample. The formula is:

```
triggered = (1 - alpha) * image + alpha * pattern
```

Because alpha is small (0.10-0.20), the trigger is nearly invisible to human inspection. This
makes the attack harder to detect via manual dataset review.

**Implementation:**
- Pattern: fixed random noise pattern generated from a seed (or loaded from file)
- Blended uniformly across the entire image rather than a localised patch
- Key ablation variable: alpha (0.05, 0.10, 0.15, 0.20, 0.30)

**Why it is harder to detect:** No obvious patch or artefact. The modification is distributed
across all pixels at low intensity. Standard visual inspection would not flag these samples.

---

### 5.3 Label-Consistent Attack (Turner et al., 2019)

The label-consistent attack is the most sophisticated of the three. It does NOT change the label
of poisoned samples. Instead, it uses projected gradient descent (PGD) adversarial perturbation
combined with a visible trigger to cause the model to associate the trigger with the target class
even without label modification.

**Implementation:**
- Requires a pretrained clean model
- For each poisoned sample: apply a base trigger (BadNets patch) then apply PGD perturbation
  (epsilon=8/255, 10 steps) pushing the model toward the target class prediction
- Labels remain unchanged in the training data
- Only the image content is modified

**Why it is stealthy:** Because labels are not changed, label-based anomaly detection and many
supervised defence methods cannot flag these samples. The attacker relies on the model learning
the trigger-target correlation from the gradient structure of the perturbation.

## 6. Defence Techniques

All defences are evaluated on the same poisoned models. The key metrics are clean accuracy (CA)
after defence and attack success rate (ASR) after defence. A good defence achieves low ASR
reduction in CA.

### 6.1 Fine-Pruning (Liu et al., 2018)

Fine-pruning exploits the observation that backdoor neurons are dormant on clean inputs but
activate strongly when a trigger is present.

**Process:**
1. Run the clean validation set through the model and record mean activation per neuron per layer
2. Prune the neurons with the lowest mean activations (bottom 30% by default)
3. Fine-tune the remaining network on clean data for 10 epochs to recover accuracy

**Why it works:** Backdoor behaviour is often concentrated in a small set of neurons. Removing
the least-used neurons (on clean data) disproportionately disrupts the backdoor pathway while
minimally affecting clean performance.

**Limitation:** If the backdoor is distributed across many neurons, pruning may not reduce ASR
significantly.

---

### 6.2 Activation Clustering (Chen et al., 2019)

Activation clustering detects poisoned samples in the training set by analysing the structure of
penultimate layer activations.

**Process:**
1. Extract penultimate layer features for all training samples, grouped by class
2. Apply PCA (10 components) to reduce dimensionality
3. Apply k-means (k=2) to each class cluster
4. If the silhouette score exceeds a threshold (indicating two distinct clusters), flag the
   smaller cluster as suspected poisoned samples
5. Remove or quarantine flagged samples

**Why it works:** Triggered samples form a distinct cluster in feature space because the trigger
is a strong feature that dominates the representation. Clean and poisoned samples of the same
class separate in the penultimate layer.

**Limitation:** For subtle triggers (blended injection at low alpha), the cluster separation may
be weak and the silhouette score may not exceed the threshold.

---

### 6.3 Spectral Signatures (Tran et al., 2018)

Spectral signatures uses singular value decomposition (SVD) to detect outlier samples in feature
space.

**Process:**
1. Extract penultimate layer features per class
2. Centre the features (subtract class mean)
3. Compute SVD of the centred feature matrix
4. Score each sample as the squared projection onto the top singular vector
5. Flag the top epsilon fraction of samples per class as suspected poisoned

**Why it works:** Backdoor patterns introduce a structured perturbation that aligns with a
low-rank component of the feature matrix. The top singular vector captures this direction and
poisoned samples project strongly onto it.

**Advantage over activation clustering:** Does not require choosing a clustering threshold.
The epsilon parameter directly controls the false positive rate.

---

### 6.4 Neural Cleanse (Wang et al., 2019)

Neural Cleanse reverse-engineers the minimal trigger pattern that would cause a model to predict
each class, then uses outlier detection to identify the backdoored class.

**Process:**
1. For each of the 43 classes, optimise a mask M and pattern P to minimise:
   `loss = CE(model(x * (1-M) + P * M), class) + lambda * ||M||_1`
   The L1 term penalises large masks, pushing toward the smallest trigger that achieves
   misclassification.
2. Collect the L1 norm of M for each class (the trigger norm)
3. Apply median absolute deviation (MAD) outlier detection across the 43 norms
4. The class with an anomalously small trigger norm is flagged as backdoored
5. Optionally unlearn the backdoor by fine-tuning on triggered inputs with random labels

**Why it works:** For a backdoored class, the optimisation finds the actual embedded trigger
(or something close to it) because it is the shortest path to misclassification. Clean classes
require much larger perturbations.

**Limitation:** Computationally expensive (43 optimisation runs). The 500-step limit per class
is a practical compromise. Also may struggle with spatially distributed triggers (blended).

## 7. Implementation Process

### 7.1 Reference Repository

The starting point was the TensorFlow-based GTSRB implementation at
github.com/hparik11/German-Traffic-Sign-Recognition. It provided:
- A working LeNet architecture for GTSRB
- The pickled dataset format (features, labels arrays)
- Preprocessing pipeline: grayscale, histogram equalisation, normalisation, augmentation
- Baseline performance target (~97% test accuracy)

This was ported entirely to PyTorch to align with the project specification and to enable
PyTorch-native features like autograd (needed for PGD in LabelConsistentAttack and for
NeuralCleanse optimisation).

### 7.2 Module Design Decisions

**Separation of concerns:** Attacks, defences, data, models, and evaluation are each isolated
into their own modules with abstract base classes (BaseAttack, BaseDefense). This allows any
attack to be paired with any defence in the pipeline without code changes.

**PoisonedDataset design:** The PoisonedDataset tracks which indices are poisoned at construction
time. This is critical for computing ASR correctly (ASR is measured only on non-target samples
from the test set with trigger applied) and for defence evaluation (detection_rate requires
knowing the true poison set).

**get_penultimate_features:** The LeNet model exposes this method directly rather than relying
on hooks. This ensures all three feature-based defences (FinePruning, ActivationClustering,
SpectralSignatures) call the same code path and operate on the same representation.

**YAML configuration:** All experiment parameters are externalised to YAML files. This enables
the ablation sweep to vary a single parameter at a time without code changes, and ensures all
results are reproducible from the config alone.

### 7.3 Ablation Study Design

The ablation study varies one parameter at a time while holding the others fixed. Four axes:

| Axis | Values swept | Fixed values |
|---|---|---|
| Poison rate | 1%, 5%, 10%, 20% | trigger_size=3, target=14 (Stop) |
| Trigger size | 2, 3, 5, 7 px | poison_rate=10%, target=14 |
| Target class | 0, 14, 33 | trigger_size=3, poison_rate=10% |
| Alpha (blended) | 0.05, 0.10, 0.15, 0.20, 0.30 | poison_rate=10%, target=14 |

For each configuration, all four defences are applied and evaluated, producing a grid of results.

Target classes were chosen to represent different levels of class imbalance:
- Class 0 (20 km/h speed limit): low frequency in training set
- Class 14 (Stop sign): medium frequency, high real-world safety significance
- Class 33 (Turn right ahead): medium frequency

### 7.4 Experiment Flow

```
train_clean.py          Train clean LeNet, save checkpoint
train_poisoned.py       Train poisoned LeNet (one of 3 attacks), save checkpoint and metrics
run_defense.py          Load poisoned checkpoint, apply one defense, report CA and ASR
run_ablation.py         Full parameter sweep across axes above
run_full_pipeline.py    End-to-end: clean -> all attacks -> all defenses -> summary table
```

All results are written to CSV via ResultsReporter and printed as ASCII tables.

## 8. Evaluation Metrics

**Clean Accuracy (CA):** Fraction of clean test samples correctly classified. Target: >95%.

**Attack Success Rate (ASR):** Fraction of triggered non-target test samples classified as the
target class. A high ASR indicates a successful attack. Target for BadNets at 10% poison rate:
>80% (consistent with Gu et al., 2019).

**CA drop after defence:** CA_before - CA_after. Should be small (<3%) for a good defence.

**ASR reduction after defence:** ASR_before - ASR_after. Should be large (>50%) for a good
defence.

**Detection rate:** Precision, recall, and F1 score comparing the set of flagged samples against
the true poison_indices from PoisonedDataset. Used to evaluate ActivationClustering and
SpectralSignatures as sample-level detectors.

---

## 9. Sanity Checks

Before drawing conclusions from results, the following checks are run:

1. Clean CA >95%: if not, preprocessing or architecture has a bug
2. Visual trigger inspection: plot 10 triggered samples per attack before training to confirm
   triggers are applied correctly
3. Poison count assertion: len(poison_indices) / len(dataset) == poison_rate within tolerance
4. ASR floor for BadNets at 10%: >80% expected; if lower, trigger injection or labelling is wrong
5. Fine-pruning identity: prune_rate=0 must leave CA unchanged
6. NeuralCleanse class detection: must identify the correct target class for BadNets
7. Reproducibility: two runs with seed=42 must produce identical CSV results

---

## 10. Limitations and Future Work

- LeNet is a shallow architecture. Results may differ on ResNet or VGG models, which are more
  commonly used in production TSR systems.
- NeuralCleanse requires 43 optimisation runs and is slow. The 500-step limit is a practical
  compromise that may reduce detection accuracy for subtle triggers.
- The label-consistent attack requires a pretrained clean model, which may not always be
  available in a realistic threat model.
- Physical-world evaluation (printing triggers on real signs and photographing them) is outside
  the scope of this project but would strengthen the real-world relevance of the findings.
- Only digital triggers are studied here. Semantic or natural language triggers (e.g. specific
  lighting conditions) are not addressed.
