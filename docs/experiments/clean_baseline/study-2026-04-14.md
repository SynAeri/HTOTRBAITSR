# Clean Baseline Study
2026-04-14

## Process Log

### Step 1: Environment Setup
- Created Python 3.13 venv at .venv/
- Installed dependencies via pip into venv (torch 2.11.0, torchvision 0.26.0, numpy 2.4.4,
  scikit-learn 1.8.0, opencv python headless 4.13.0)
- NixOS required shell.nix to expose libstdc++, libz, libGL, glib for C extension loading
- Confirmed dataset loaded correctly: train (34,799), valid (4,410), test (12,630)

### Step 2: Dataset Analysis
- Discovered significant class imbalance: 11.2x ratio between most and least frequent class
  - Most frequent: class 2 (50 km/h) with 2,010 samples
  - Least frequent: class 0 (20 km/h) with 180 samples
  - Mean class count: 809, Std: 619
- Images are 32x32 RGB uint8, sourced from real world traffic sign photographs
- Preprocessing pipeline selected: histogram equalisation per channel (addresses lighting
  variation), normalisation to mean 0.5 std 0.5, random rotation and affine augmentation
  during training only

### Step 3: Model Architecture
- LeNet CNN ported from TensorFlow reference (hparik11/German Traffic Sign Recognition)
  to PyTorch
- Architecture: Conv(3,32,5) > ReLU > MaxPool > Conv(32,64,5) > ReLU > MaxPool >
  FC(4096,512) > Dropout(0.5) > FC(512,256) > Dropout(0.5) > FC(256,43)
- Total parameters: approximately 2.4M
- Input: 3x32x32, Output: 43 class logits

### Step 4: Training
- Optimiser: Adam lr=0.001, weight decay=1e-4
- Scheduler: CosineAnnealingLR over 30 epochs
- Batch size: 64, seed: 42
- Device: CPU (no GPU available)
- Train set: 27,839 samples (80% of train.p after stratified split)
- Val set: 6,960 samples (20% of train.p)

### Step 5: Results
| Metric | Value |
|---|---|
| Best val CA (epoch 28) | 99.87% |
| Final test CA | 96.70% |

The gap between val CA and test CA is expected. The val set is a stratified holdout from
the same distribution as training, while the test set contains more real world variation
in image quality, angle, and lighting. 96.70% exceeds the target of 95%.

### Step 6: Checkpoint
Saved to checkpoints/clean_lenet.pth with metadata epoch=28, val_acc=0.9987.
This checkpoint is used as the reference model for the label consistent attack (PGD
perturbation requires a pretrained model) and as the clean baseline for all defence
evaluations.

- 96.7% test CA is the CA ceiling: any poisoned model deviating significantly below this
  is leaking attack information through degraded accuracy
